# Data Model: Digital Twin (Gazebo & Unity)

**Feature**: 001-digital-twin-gazebo-unity
**Date**: 2025-12-09
**Input**: Feature specification analysis

## Key Entities

### 1. Digital Twin
- **Description**: Physics-accurate virtual representation of a humanoid robot that exists simultaneously in both Gazebo (physics) and Unity (visualization) environments
- **Attributes**:
  - id: unique identifier for the digital twin instance
  - name: human-readable name for the digital twin
  - creation_date: timestamp when the digital twin was created
  - status: current operational status (active, paused, error)
  - physics_state: synchronized state in Gazebo environment
  - visualization_state: synchronized state in Unity environment
  - configuration: settings for physics, rendering, and sensor simulation
- **Relationships**: Contains one or more Humanoid Robot Models
- **Validation**: Must have valid physics and visualization states

### 2. Humanoid Robot Model
- **Description**: Virtual robot entity that serves as the core object in the digital twin, supporting physics simulation, visualization, and sensor data generation
- **Attributes**:
  - id: unique identifier for the robot model
  - name: human-readable name for the robot
  - model_file_path: path to the robot's URDF/SDF/Mesh file
  - joint_configurations: array of joint positions and limits
  - sensor_attachments: list of sensors attached to the robot
  - physical_properties: mass, center of mass, inertia tensor
  - kinematic_chain: joint hierarchy and transformation matrices
- **Relationships**: Belongs to a Digital Twin, contains multiple Sensors
- **Validation**: Must have valid URDF/SDF representation, joints within limits

### 3. Sensor Simulation Pipeline
- **Description**: System component that generates realistic sensor outputs (LiDAR, depth, IMU) based on the robot's position and environment in the digital twin
- **Attributes**:
  - id: unique identifier for the pipeline
  - digital_twin_id: reference to the parent digital twin
  - sensor_types: list of active sensor types (LiDAR, depth, IMU, etc.)
  - output_format: data format for each sensor type
  - update_frequency: frequency at which sensor data is updated
  - noise_model: parameters for adding realistic noise to sensor data
  - calibration_data: calibration parameters for each sensor
- **Relationships**: Belongs to a Digital Twin, processes data from multiple Sensors
- **Validation**: Must have valid sensor types, frequencies within acceptable range

### 4. Educational Content
- **Description**: Documentation, examples, and tutorials designed for students learning physical-AI humanoid simulations
- **Attributes**:
  - id: unique identifier for the content piece
  - title: human-readable title of the content
  - content_type: type of content (tutorial, example, documentation)
  - target_audience: intended audience level (beginner, intermediate, advanced)
  - module: associated module (Gazebo Physics, Unity Rendering, Sensor Simulation)
  - difficulty_level: relative difficulty (1-5 scale)
  - estimated_duration: time needed to complete the content
  - prerequisites: list of required knowledge or skills
  - reproducible_examples: code and configuration files for the example
  - citations: list of sources and references (IEEE format)
- **Relationships**: May reference one or more Digital Twins or Robot Models
- **Validation**: Must have valid citations, reproducible examples must be tested

## Additional Supporting Entities

### 5. Gazebo Environment
- **Description**: Physics simulation environment with gravity, collision detection, and environmental interactions
- **Attributes**:
  - id: unique identifier for the environment
  - name: human-readable name for the environment
  - gravity: gravitational force vector
  - collision_meshes: list of collision objects in the environment
  - lighting: lighting configuration for the physics simulation
  - physics_engine: selected physics engine (ODE, Bullet, etc.)
  - time_step: simulation time step configuration
- **Relationships**: Contains one or more Digital Twins

### 6. Unity Scene
- **Description**: High-fidelity visualization environment for human-robot interaction scenarios
- **Attributes**:
  - id: unique identifier for the scene
  - name: human-readable name for the scene
  - lighting_config: lighting and shadow settings
  - camera_config: camera positions and settings
  - rendering_quality: quality settings for the scene
  - interactive_elements: list of interactive objects in the scene
  - animation_clips: available animation sequences
- **Relationships**: Contains visualization of Digital Twins and Robot Models

### 7. Sensor Data Output
- **Description**: Realistic sensor data generated by the simulation for perception tasks
- **Attributes**:
  - id: unique identifier for the data output
  - sensor_type: type of sensor (LiDAR, depth, IMU, etc.)
  - timestamp: time when the data was captured
  - raw_data: raw sensor readings
  - processed_data: processed sensor data (point clouds, depth maps, etc.)
  - coordinate_frame: reference frame for the data
  - quality_metrics: measures of data quality and accuracy
- **Relationships**: Generated by Sensor Simulation Pipeline
- **Validation**: Data must be in correct format for perception algorithms
